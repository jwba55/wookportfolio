클라우드 서버 배포
Azure Web App for Containers + Docker Hub + GitHub Actions 기반 배포

1. Azure Web App 생성
	1) Azure Portal 로그인
	2) Web App 생성
	3) Publish: Docker Container 선택
	4) OS: Linux 선택
	5) Web App 이름 지정
	6) Docker Hub 이미지 연결
		- 예) wlwl45wl/wookportfolio:latest

2. Azure 환경 변수 설정
	Azure Portal > Web App > Environment variables 에서 아래 값 추가
		- WEBSITES_PORT=8089
		- SPRING_PROFILES_ACTIVE=prod

3. GitHub Secrets 설정
	저장소 > Settings > Secrets and variables > Actions 에서 아래 값 등록
		- AZURE_CREDENTIALS
		- AZURE_WEBAPP_NAME
		- DOCKER_USERNAME
		- DOCKER_PASSWORD

4. GitHub Actions 자동 배포
	main 브랜치에 push 하면 아래 순서로 자동 배포
		1) Maven 빌드
		2) Docker 이미지 빌드
		3) Docker Hub push
		4) Azure 로그인
		5) Azure Web App 배포

5. 접속 확인
	Azure Web App 기본 주소로 접속
		예) https://<웹앱이름>.azurewebsites.net

6. 참고
	- Azure Web App은 기본 HTTPS를 제공하므로 별도의 Nginx, Certbot, DuckDNS 설정이 필수는 아님
	- 커스텀 도메인이 필요하면 Azure의 Custom domains 기능으로 추가 가능
